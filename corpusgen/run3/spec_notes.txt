## beacon (subtype 8)
Beacon management frame, Frame Control Type=0 (management), Subtype=8. Periodically broadcast by an AP (or sent in an IBSS) to announce the BSS, carry timing (Timestamp), the Beacon Interval (in TUs), and Capability Information, followed by IEs describing the BSS. Fixed-field portion (12 octets) baseline = Timestamp(8)=0000000000000000, Beacon Interval(2)=6400 (100 TU, little-endian), Capability Information(2)=0100 (ESS bit). SSID and Supported Rates are mandatory; DS Parameter Set, TIM, Country, RSN, and Extended Supported Rates are optional/situational. Multi-octet fixed fields are little-endian per 802.11 MAC conventions. Beacons are unprotected management frames sent regardless of association state.

- IE 0 SSID [mandatory]: Element ID 0; SSID 0-32 octets; baseline is ASCII 'owfuzzai'. Empty value (length 0) denotes a wildcard/broadcast SSID. Mandatory in Beacon per the Beacon subtype description.
- IE 1 Supported Rates [mandatory]: Element ID 1; 1-8 octets, one rate per octet in units of 500 kb/s; MSB set marks a Basic Rate. Baseline lists 1(B),2(B),5.5(B),11(B),6,9,12,18 Mb/s. Rates beyond 8 spill into Extended Supported Rates (ID 50). Mandatory per spec.
- IE 3 DS Parameter Set [optional]: Element ID 3; exactly 1 octet = current operating channel. Baseline channel 6. Present for DSSS/HR/ERP/HT 2.4 GHz PHYs to advertise the operating channel.
- IE 5 TIM [optional]: Element ID 5; Traffic Indication Map. Octets: DTIM Count (00), DTIM Period (01), Bitmap Control (00), Partial Virtual Bitmap (00). Carried in Beacons from an AP to signal buffered group/individual traffic.
- IE 7 Country [optional]: Element ID 7; Country String (3 octets, e.g. 'US' + outdoor/indoor/any flag 0x20) followed by one or more regulatory triplets (First Channel=01, Number of Channels=0b, Max Tx Power=1e dBm). Advertises regulatory domain constraints.
- IE 48 RSN [optional]: Element ID 48; Robust Security Network. Version (0100), Group Data Cipher Suite (00-0f-ac-04 CCMP), Pairwise Cipher Suite Count (0100) + suite (00-0f-ac-04 CCMP), AKM Suite Count (0100) + suite (00-0f-ac-02 PSK), RSN Capabilities (0000). Present when the BSS advertises an RSNA.
- IE 50 Extended Supported Rates [optional]: Element ID 50; same encoding as Supported Rates (500 kb/s units, MSB = Basic Rate). Used when more than 8 rates are supported; baseline lists 24,36,48,54 Mb/s. Optional per spec.

## probe_request (subtype 4)
Probe Request, management subtype 4. Sent by a station performing active scanning to solicit Probe Responses from BSSs. It has no fixed fields; the frame body is purely information elements. SSID (wildcard for broad discovery or a specific SSID for directed scan) and Supported Rates are mandatory; Extended Supported Rates is optional. The MAC header carries broadcast or directed DA, the scanning STA's SA, and broadcast or directed BSSID.

- IE 0 SSID [mandatory]: Element ID 0, 0-32 octets. Value shown is ASCII 'owfuzzai'. A zero-length SSID ('') denotes a wildcard/broadcast probe used to discover all BSSs. Per Beacon/Probe IE definition; mandatory in Probe Request.
- IE 1 Supported Rates [mandatory]: Element ID 1, 1-8 octets. Each octet encodes a rate in 500 kbps units; high bit (0x80) marks a Basic (mandatory) rate. Example encodes 1,2,5.5,11 (basic) plus 6,9,12,18 Mbps. Mandatory in Probe Request.
- IE 50 Extended Supported Rates [optional]: Element ID 50, used when more than 8 rates must be advertised (overflow from Supported Rates). Example encodes 24,36,48,54 Mbps. Optional in Probe Request.

## authentication (subtype 11)
802.11 Authentication management frame, Frame Control Type=0 (management), Subtype=11. Frame body = three fixed fields in order, all 2 octets, little-endian: Authentication Algorithm Number (0=Open System, 1=Shared Key, 3=SAE), Authentication Transaction Sequence Number, and Status Code. Baseline fixed value 000001000000 encodes Open System (algorithm 0), sequence 1, status 0 (success). Authentication establishes identity between a STA and AP and is a prerequisite to Association; Open System completes in a 2-frame exchange (seq 1 request, seq 2 response), Shared Key uses a 4-frame challenge/response exchange in which the AP-generated Challenge Text element (ID 16) appears in frames 2 and 3, and SAE carries its own fields/elements. The Challenge Text IE is the only information element defined for this frame in the provided spec, and only conditionally.

- IE 16 Challenge Text [conditional]: Present ONLY in Shared Key authentication, transaction sequence frames 2 and 3 (absent in Open System and SAE). The value is an opaque random challenge string (per 802.11 the challenge text is 128 octets when generated by the AP); the bytes shown are a representative truncated example. For Open System auth (algorithm 0), no IEs are present at all.

## assoc_request (subtype 0)
Association Request (management subtype 0). Sent by a STA to join a BSS after successful authentication. Frame body = fixed fields Capability Information (2 octets) and Listen Interval (2 octets), shown as 1104 0a00, followed by IEs in order: SSID and Supported Rates (mandatory), RSN (conditional, present for RSNA), Extended Capabilities (optional). MAC header carries Type=0 (management); Address 1=DA(AP), Address 2=SA(STA), Address 3=BSSID.

- IE 0 SSID [mandatory]: Element ID 0; 0-32 octets. Value shown is ASCII 'owfuzzai'. Identifies the target BSS the STA is joining.
- IE 1 Supported Rates [mandatory]: Element ID 1; 1-8 octets. Each octet encodes a rate in 500 kb/s units; MSB set marks a Basic (mandatory) rate. Shown set = 1,2,5.5,11 Mb/s basic plus 6,9,12,18 Mb/s.
- IE 48 RSN [conditional]: Element ID 48; present for RSNA (robust security network association). Value = Version(0100) | Group Cipher(000fac04=CCMP) | Pairwise Count(0100) | Pairwise(000fac04=CCMP) | AKM Count(0100) | AKM(000fac02=PSK) | RSN Capabilities(0000).
- IE 127 Extended Capabilities [optional]: Element ID 127; optional. Bitmask of extended capability flags; baseline shown advertises a minimal/representative set.
